## Errors Found (249 total)

### ✅ FIXED — 33 incorrect pages found in sitemap.xml
**Root cause:** `scripts/generate-sitemap.js` was falling back to bare numeric IDs (e.g. `/cars/6515171`) for cars without a `slug` field. These pages either 404 or redirect, making them "incorrect" per Semrush.  
**Fix applied (18 Feb 2026):** Generator now skips cars with no slug entirely. Sitemap regenerated — 131 valid URLs, zero bare-ID entries.  
**Files changed:** `scripts/generate-sitemap.js`, `public/sitemap.xml`

---

### ✅ FIXED — 156 hreflang conflicts within page source code
**Root cause:** `components/Seo.tsx` was building all hreflang `href` values pointing to the homepage root (`siteOrigin/` or `siteOrigin/es/`) regardless of the actual page path. Every page therefore had hreflang tags conflicting with every other page.  
**Fix applied (18 Feb 2026):** Hreflang hrefs now use the actual page path extracted from the canonical `url` prop, constructing correct locale-prefixed URLs (e.g. `https://autogo.pt/es/cars/bmw-ix-...` for Spanish). Added `x-default` pointing to the pt-PT canonical.  
**Files changed:** `components/Seo.tsx`

---

### ✅ FIXED — 33 issues with incorrect hreflang links
**Root cause:** Same as above — all non-pt-PT hreflang tags were pointing to bare locale root paths instead of the current page's locale-equivalent path. No `x-default` was emitted.  
**Fix applied (18 Feb 2026):** Corrected in same `Seo.tsx` update. `x-default` tag now included in every page's `<head>`.  
**Files changed:** `components/Seo.tsx`

---

### ✅ FIXED — 18 structured data items are invalid
**Root cause (4 sub-issues in `pages/cars/[slug].tsx`):**
1. `@type: 'Vehicle'` — not a valid Google Rich Result type. Should be `Car`.
2. `mileageFromOdometer` set as a bare number — Schema.org requires `QuantitativeValue` object with `unitCode: 'KMT'`.
3. `offers.price: null` for "Sob Consulta" cars — invalid in schema (price must be a number or omitted).
4. No `vehicleModelDate` field (manufacture year was missing from structured data).

**Fix applied (18 Feb 2026):**
- Changed `@type` to `'Car'`
- `mileageFromOdometer` now emits `{ "@type": "QuantitativeValue", "value": N, "unitCode": "KMT" }`
- Null-price offers now omit the `price` field and include `description: 'Preço sob consulta'` instead
- Added `vehicleModelDate` from `car.year`
- Added `fuelType` and `vehicleEngine` fields for richer data
- Added `seller` (`AutoDealer`) inside every `Offer`

**Files changed:** `pages/cars/[slug].tsx`

## ⚠️ Warning Fixed (912 issues)

### ✅ FIXED — 912 issues with blocked internal resources in robots.txt
**Root cause:** `Disallow: /_next/` blocked ALL Next.js static assets — JS bundles, CSS, fonts, images. Semrush and Googlebot couldn't render pages, flagging 912 resource-load warnings across all crawled pages.  
**Fix applied (18 Feb 2026):** Replaced blanket `/_next/` disallow with:
- `Allow: /_next/static/` — explicitly permits all build assets
- `Disallow: /_next/server/` — blocks internal SSR-only paths
- `Disallow: /_next/image?*` — blocks the on-demand image resizing endpoint (not needed by crawlers)

**Files changed:** `public/robots.txt`
